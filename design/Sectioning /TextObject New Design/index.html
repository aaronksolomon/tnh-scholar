    TextObject System Design Document

    1. Overview and Purpose

    The TextObject system provides a robust way to manage sectioned text content with associated metadata. It serves two key purposes: internal text processing with rich metadata support, and integration with AI services through a simplified format. The design emphasizes simplicity, standardization, and effective AI context handling.

    2. Core Components

    2.1 Response Format (API Layer)

    The response format provides a simplified structure optimized for AI interaction:

    class LogicalSection(BaseModel):
    """Represents a logical division of text content."""
    start_line: int = Field(..., description="Starting line number of section (inclusive)")
    title: str = Field(..., description="Title describing section content")

class TextMetadataFormat(BaseModel):
    """Simplified metadata format for AI processing."""
    metadata_summary: str = Field(
        ...,
        description="Available metadata in human-readable format"
    )
    context: str = Field(
        ...,
        description="Rich contextual information for AI understanding"
    )

class TextObjectFormat(BaseModel):
    """Complete format for API interactions."""
    metadata: TextMetadataFormat
    language: str = Field(..., description="ISO 639-1 language code")
    sections: List[LogicalSection]

    2.2 Internal Representation

    The internal system uses a richer structure based on Dublin Core standards:

    @dataclass
class TextMetadata:
    """Rich metadata container following Dublin Core standards."""
    # Core Dublin Core elements
    title: str
    creator: List[str]
    subject: List[str]
    description: str
    publisher: Optional[str] = None
    contributor: List[str] = field(default_factory=list)
    date: Optional[str] = None
    type: str
    format: str
    identifier: Optional[str] = None
    source: Optional[str] = None
    language: str

    # Additional contextual information
    context: str = field(default="")
    additional_info: Dict[str, Any] = field(default_factory=dict)

    2.3 Primary TextObject Class

    class TextObject:
    """
    Main class for managing sectioned text content with metadata.
    """
    def __init__(
        self,
        numbered_text: NumberedText,
        language: str,
        sections: List[LogicalSection],
        metadata: TextMetadata
    ):
        self.content = numbered_text
        self.language = language
        self.sections = sorted(sections, key=lambda x: x.start_line)
        self.metadata = metadata
        self.total_lines = numbered_text.size
        self._validate()

    3. Key Design Decisions

    3.1 Metadata Handling

    The system employs a dual approach to metadata:

    1. Internal Storage
    2. Uses Dublin Core standards for structured metadata
    3. Maintains complete metadata in standardized format

    4. Supports optional fields and additional information

    5. API Representation

    6. Converts metadata to human-readable string format
    7. Separates factual metadata from contextual information
    8. Optimizes for AI processing and generation

    3.2 Section Management

    The system uses a simplified section representation:

    • Sections defined only by start line
    • Implicit end lines (next section's start - 1)
    • Guaranteed contiguous coverage
    • Ordered storage and validation

    3.3 Content Integration

    Content management is delegated to NumberedText class:

    • Clean separation of concerns
    • Efficient text storage and access
    • Section-aware interface

    4. Implementation Details

    4.1 Format Conversion

    class TextObject:
    def to_response_format(self) -> TextObjectFormat:
        """Convert to API format."""
        metadata_summary = self._format_metadata_string()

        return TextObjectFormat(
            metadata=TextMetadataFormat(
                metadata_summary=metadata_summary,
                context=self.metadata.context
            ),
            language=self.language,
            sections=self.sections
        )

    @classmethod
    def from_response_format(cls, 
                           text: NumberedText,
                           response: TextObjectFormat) -> "TextObject":
        """Create from API response."""
        metadata = cls._parse_metadata_string(
            response.metadata.metadata_summary,
            response.metadata.context
        )
        return cls(
            numbered_text=text,
            language=response.language,
            sections=response.sections,
            metadata=metadata
        )

    4.2 Validation

    def _validate(self) -> None:
    """Validate section integrity."""
    if not self.sections:
        raise ValueError("TextObject must have at least one section")

    # Validate section ordering
    for i, section in enumerate(self.sections):
        if section.start_line < 1:
            raise ValueError(f"Section {i}: start line must be >= 1")
        if section.start_line > self.total_lines:
            raise ValueError(f"Section {i}: start line exceeds text length")
        if i > 0 and section.start_line <= self.sections[i-1].start_line:
            raise ValueError(f"Section {i}: non-sequential start line")

    5. Usage Patterns

    5.1 Section Access

    def get_section_content(self, index: int) -> str:
    """Retrieve content for specific section."""
    start = self.sections[index].start_line
    end = (self.sections[index + 1].start_line 
           if index < len(self.sections) - 1 
           else self.total_lines + 1)
    return self.content.get_segment(start, end)

    5.2 Metadata Processing

    def _format_metadata_string(self) -> str:
    """Convert structured metadata to formatted string."""
    parts = []
    if self.metadata.title:
        parts.append(f"Title: {self.metadata.title}")
    if self.metadata.creator:
        parts.append(f"Creator(s): {', '.join(self.metadata.creator)}")
    # Add other metadata fields...
    return "\n".join(parts)
